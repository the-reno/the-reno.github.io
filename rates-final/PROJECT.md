# SOFR Monthly Roll Factor Model

## Project description

This project is a historical investment-roll simulator for SOFR-based placements.

It compares the investment factors generated by repeatedly investing at different tenors over the same historical period:

- ON
- 1M
- 2M
- 3M
- 6M

The model uses a common monthly roll schedule so that all strategies are aligned to the same maturity framework. Longer-tenor factors are displayed only when those investments reach their scheduled maturity.

The project is available as:

- A public web model at `/rates-final/`
- A CSV curve file
- Simple VBA modules for Excel

## Project target

The target is to provide a simple and transparent tool that can:

1. Simulate historical investment decisions using SOFR rates available on each placement date.
2. Compare different placement tenors using the same analysis start and final date.
3. Preserve a fixed monthly roll-day convention.
4. Adjust unavailable roll dates to the previous available curve date.
5. Show each tenor factor only when the corresponding investment matures.
6. Produce a consistent monthly table that can be reviewed, exported, or reproduced in Excel.

The model is intended for historical analysis, comparison, education, and treasury planning support. It is not intended to replace official market data, valuation systems, or transaction records.

## User inputs

The user provides:

### Final end date

The final date of the analysis.

Example:

```text
26-Jun-2026
```

### Analysis period

The number of years to look back from the final date.

Example:

```text
3 years
```

The theoretical analysis start date would be:

```text
26-Jun-2023
```

### Curve file

The model accepts a CSV file with the following headers:

```text
Date,ON,1M,2M,3M,6M
```

Example:

```text
2026-05-26,3.61244,3.65225,3.69764,3.73350,3.86235
```

Rates are stored in percentage points. Therefore:

```text
3.65225 = 3.65225%
```

The calculation converts the rate to decimal form by dividing by 100.

## Date logic

### 1. Fixed roll day

The model uses the day of the final end date as the fixed monthly roll day.

Example:

```text
Final date: 26-Jun-2026
Fixed roll day: 26
```

The theoretical monthly ladder is:

```text
26-Jun-2023
26-Jul-2023
26-Aug-2023
...
26-Jun-2026
```

### 2. Curve dates are the master calendar

Every date used by the model must exist in the curve file.

If a theoretical roll date is unavailable, the model uses the previous available curve date.

Example:

```text
Theoretical roll date: 26-Aug-2023
Previous available curve date: 25-Aug-2023
Adjusted roll date: 25-Aug-2023
```

The model never moves an unavailable date forward.

### 3. Common monthly schedule

All tenors use the same monthly maturity anchors.

Example:

| Month | Monthly Start | Monthly End |
|---:|---|---|
| 1 | 26-Jun-2023 | 26-Jul-2023 |
| 2 | 26-Jul-2023 | 25-Aug-2023 |
| 3 | 25-Aug-2023 | 26-Sep-2023 |

The Start and End columns describe the monthly reporting interval. A longer-tenor factor may have an investment start date earlier than the monthly Start shown in the row.

## Tenor maturity logic

Each tenor is shown only when its investment matures.

### ON

Shown every month.

The current version uses the ON rate available on the monthly placement start date and applies it over the monthly interval.

### 1M

Shown every month.

The investment starts on the previous monthly roll date and matures on the current roll date.

### 2M

Shown every second month:

```text
2, 4, 6, 8, 10, 12...
```

The start date is two roll dates before the current maturity date.

### 3M

Shown every third month:

```text
3, 6, 9, 12...
```

The start date is three roll dates before the current maturity date.

### 6M

Shown every sixth month:

```text
6, 12, 18, 24...
```

The start date is six roll dates before the current maturity date.

Blank cells mean that the relevant investment does not mature in that month.

## Rate selection logic

The rate is always taken from the investment start date.

Example for a 2M investment:

```text
Investment start: 26-Jun-2023
Investment maturity: 25-Aug-2023
```

The model uses the 2M rate from the curve row dated 26-Jun-2023.

The maturity-date rate is not used for that investment.

## Factor calculation

The model uses simple interest on an Actual/360 basis.

```text
Factor = 1 + Rate × Days / 360
```

Because CSV rates are expressed in percentage points:

```text
Factor = 1 + (Rate / 100) × Days / 360
```

Where:

```text
Days = Maturity Date - Investment Start Date
```

Calendar days are used between the adjusted curve dates.

### Example

```text
Start date: 26-Dec-2025
End date: 26-Jun-2026
Days: 182
6M rate on start date: 3.58%
```

```text
Factor = 1 + 0.0358 × 182 / 360
       = 1.0180989
       = 1.018099
```

## Validation

The model verifies:

- The curve file contains all required columns.
- Dates are valid and can be sorted chronologically.
- The curve covers the requested analysis period.
- The final date can be adjusted to an available curve date.
- Every monthly roll date can be adjusted to a previous available curve date.
- The selected tenor rate exists on the investment start date.
- The maturity date is later than the investment start date.

The web page displays:

- Curve row count
- Earliest curve date
- Latest curve date
- Adjusted analysis start date
- Adjusted final date
- Number of monthly output rows

## Key assumptions

### Previous-date adjustment

Unavailable dates always move to the previous available curve date.

### Common maturity framework

All strategies use the same monthly maturity anchors.

### Simple interest

Term factors use simple Actual/360 interest.

### ON simplification

The current ON factor is a simplified monthly placement using the ON rate observed at the monthly start date.

A true overnight reinvestment strategy would require daily compounding:

```text
Monthly ON Factor = Product of all daily ON accrual factors
```

That enhancement can be added separately if required.

## Process summary

```text
1. Read the final date and analysis years.
2. Calculate the theoretical analysis start date.
3. Build the fixed-day monthly roll schedule.
4. Move unavailable dates to the previous available curve date.
5. Use one common monthly maturity schedule for all tenors.
6. Identify which tenors mature in each month.
7. Read each tenor rate from its investment start date.
8. Calculate the Actual/360 simple-interest factor.
9. Leave non-maturity tenor cells blank.
10. Display the monthly comparison table.
```
