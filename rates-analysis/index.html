Corrected logic

  • Every calendar date is a possible cash-availability scenario.
  • The actual start is the first available curve date on or after the scenario date.
  • The actual maturity is the latest curve date on or before the tenor target.
  • The same-tenor maturity rate is compared with the same-tenor start rate.
  • Reset volatility remains in basis points without annualization or tenor normalization.
  • The efficient frontier uses aligned complete-month strategy returns and remains separate from reinvestment reset risk.
  • Every frontier point includes a rank, segment, description, allocation summary, WAM and liquidity profile.

Use in Excel

  1. Download the version 3 VBA module above.
  2. Open a blank macro-enabled workbook.
  3. Import Rates_Analysis_Final_v3.bas through the VBA editor.
  4. Run CreateBlankRatesModel.
  5. Enter the inputs and paste the curve with headers Date | ON | 1M | 2M | 3M | 6M.
  6. Run BuildRatesAnalysisModel.
Rates are entered as percentage points: 1.54 means 1.54%. The rolling strategy uses ACT/360. The daily reset analysis uses next-available start and prior-available maturity dates.
