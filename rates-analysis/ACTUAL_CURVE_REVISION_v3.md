## Revised date logic

For every calendar scenario date:

- **Actual start:** first curve date on or after the scenario date.
- **Target maturity:** scenario date plus the selected tenor, month-end aware.
- **Actual maturity:** latest curve date on or before the target maturity.
- **Reset:** same-tenor maturity rate minus same-tenor start rate, expressed in basis points.
- **ON:** actual start to the next available curve date.

The daily reset observations overlap intentionally because every calendar date is a separate cash-availability scenario.
