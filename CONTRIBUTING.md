# Contributing

1. Create a short-lived branch from `main`.
2. Put public website files under `docs/` only.
3. Preserve existing public URLs unless a redirect plan is included.
4. Do not commit credentials, personal records, private financial data, or unpublished workbooks.
5. Run `python scripts/check_site.py` and preview the site locally.
6. Open a pull request describing content, URL, privacy, and deployment impact.

See `project-docs/` for the complete project conventions.
