# Content guide

## Add a page

1. Choose the owning section under `docs/`.
2. Use a lowercase kebab-case filename, such as `systems-thinking.html`.
3. Copy the metadata and privacy directives from a neighboring page.
4. Use root-relative internal links, such as `/Science/systems-thinking.html`.
5. Add the page to its section index and, only when appropriate, the homepage.
6. Run the validator and browser preview.

## Add a section

Create `docs/<section>/index.html`. Keep its images, scripts, and styles within that directory until they become shared resources.

## Content rules

- Preserve the established visual voice and navigation behavior.
- Supply descriptive page titles and one visible `h1` for new long-form pages.
- Prefer semantic HTML and keyboard-accessible controls.
- Compress images and provide meaningful alternative text.
- Do not expose drafts, source datasets, spreadsheets, credentials, or personal records.
- Document any new external font, script, analytics tool, form, or embedded service in the privacy policy before deployment.
